正睿 20 连的第一套

作为开场的第一场似乎比赛本身手感不错？但是状态不好所以有点摆xd

A

这个东西居然可以 O(1)O(1)O(1)，比较惊讶。

赛时犯了一堆大唐（

注意到这个操作本质上是加上一个长度 a+ba + ba+b 的数在最后面然后右移 a+ba + ba+b。

其中这个数字有 aaa 个 1 和 bbb 个 0，那我们直接贪心选择这个数字的最大和最小值就好了。

B

这个题挺 ad-hoc 的吧，第一眼看成只有一个空位了。

如果是两个空位加上这个数据范围就应该猜测是一个 n2n ^ 2n2 或者 n3n ^ 3n3 的东西。

考虑这个大环允许我们做一个循环移位。

那么如果我们有一条不在大环上的边我们就可以交换两个相邻的位置了。

限制很松，冒泡即可。

C

场上差一点会完了，很牛的串串 ds 题。

一个很显著的结论就是直接匹配 LCP 和 LCS，剩下部分只能替换。

前者我们建立一棵前缀 Trie，后者我们将所有后缀哈希后放入桶中。

对于每一个桶，我们考虑先按照原串长度小到大排序。

然后这个东西相当于是固定了我们后缀的长度，前缀我们在字典树上找到对应长度的点并统计。

这个可以简单通过字典树 + dfn + BIT 维护

有一个小问题就是我们这个后缀长度是至少，会算重，考虑向前再看一个字符，如果相同的对数我们就不计算。

那么我们在这里就干脆开 26 颗 Trie 分别维护即可。

D

感觉在哪里见过但是还是不会的一个 dp 题。

在 bbb 序列排序后考虑这个问题是显然的，不难注意到随 RRR 增 LLL 不降。

fpf_pfp​ 表示 R=pR = pR=p 的最优解，这个东西是一个 1D1D 的转移，可以有一个分治的 log2log^2log2 数据结构（简单）做法。

考虑这怎么优化？我们做 插入 + kth 和不好做，但是删除 + kth 和极好做，排序 + 链表维护就可以拍掉这个log。

这个排序的 log 就是一开始排序然后划分下去就可以避免。

希望下一场的题也可以这么牛吧。
